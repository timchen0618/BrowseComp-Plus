You are evaluating the quality of a plan for an agentic search system. The agent follows a plan to iteratively query a search tool (a dense retriever) and reason over results until it finds the answer to a complex question from the BrowseComp benchmark.
You will be given a question and a candidate plan. Score the plan on how likely it is to lead the agent to the correct answer if faithfully executed.

Evaluate the plan along the following dimensions:

1. **Decomposition quality**: Does the plan break the question into logical, sequential substeps? Are dependencies between steps clear?
2. **Substep Clarity**: Is the goal of each substep clear? If there are higher-level goals, are they broken down to smaller subgoals for easier execution?
3. **Search query effectiveness**: Are the implied or explicit search queries realistic — i.e., would they plausibly return useful results from a real search engine? (Short, focused queries > long keyword-stuffed strings.)
4. **Critical path focus**: Does the plan prioritize the steps that are strictly necessary to reach the answer, versus spending budget on nice-to-have verification?
5. **Prioritization of Constraints**: Does the plan prioritize the most distinctive constraint so that meaningful information is more likely to be added after first few searches?
6. **Robustness to failure**: Does the plan account for what happens if a step returns no results or ambiguous results? Are there fallback strategies?
7. **State tracking**: Does the plan clearly indicate what is known, what is hypothesized, and what remains to be found?
8. **Budget awareness**: Is the plan feasible within a limited number of search calls (assume a budget of ~20 calls)?

Think step by step about the strengths and weaknesses of the plan along each dimension, then give an overall score from 1 to 5:
  1 = Very poor — unlikely to lead to the correct answer
  2 = Poor — major weaknesses that significantly reduce chance of success
  3 = Adequate — reasonable approach but with notable gaps
  4 = Good — well-structured with minor issues
  5 = Excellent — comprehensive, efficient, and robust

The output should be a JSON object with the following schema:
Output format:
```
{
    "thinking": [step-by-step thinking and rationale for giving the score],
    "score": [integer score from 1 to 5]
}
```
